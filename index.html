1. Naam aanpassen

Opdracht

Voor je begint met het aanpassen van de code, zorg dat je in de master branch zit.

  • Vervang bovenaan de pagina "Naam van student" door jouw eigen naam (id=naam)
  • Commit deze wijziging

2. Afbeeldingen toevoegen

Opdracht

Voor je begint met het aanpassen van de code, zorg dat je in de master branch zit.

  • Maak een nieuwe branch: afbeeldingen
  • Controleer dat je in deze branch zit
  • Zoek een passende afbeelding voor dit hoofdstuk (id=afbeeldingen), voeg deze afbeelding toe in de map images.
  • Pas de src aan van de afbeelding van dit hoofdstuk aan, zodat de nieuw toegevoegde afbeelding zichtbaar is.
  • Commit deze wijziging
  • Ga terug naar de master branch.
  • Merge de aanpassingen van afbeeldingen naar master.
  • Push de master branch naar github

3. Een conflict oplossen

Winnaar: Chuck Norris

Chuck Norris

  • Begin een nieuwe branch: chuck vanuit master
  • Pas de hierboven aangegeven winnaar (id=winnaar) aan naar: Chuck Norris
  • Pas de src van de afbeelding (id=winnaar-image) aan naar: images/chuck-norris.jpg
  • Commit deze wijziging.

Bruce Lee

  • Begin een nieuwe branch: bruce vanuit master
  • Pas de hierboven aangegeven winnaar (id=winnaar) aan naar: Bruce Lee
  • Pas de src van de afbeelding (id=winnaar-image) aan naar: images/bruce-lee.jpg
  • Commit deze wijziging.

Conflict!

  • Ga terug naar de master branch
  • Refresh de pagina en check of Niemand de winnaar is
  • Merge de aanpassingen van bruce naar master.
  • Controleer of Bruce lee de winnaar is (refresh de pagina).
  • Merge de aanpassingen van chuck naar master.
  • Conflict!

    Je hebt nu een conflict in de header van dit article.

  • Zorg dat Chuck Norris de winnaar is.
  • Commit de merge
  • Push de master branch naar github

4. Een conflict oplossen

Soms is er bij een conflict geen keuze, maar moet je handmatig de wijzigingen samen voegen.

Donaties: BTC 1,01

Gulle gever

  • Begin een nieuwe branch: gulle-gever vanuit master
  • De gulle gever heeft net 1 hele Bitcoin in onze Wallet gestopt.
    Pas het hierboven aangegeven donatiebedrag (id=donaties) aan naar: 1,00
  • Commit deze wijziging.

Dagobert Duck

  • Begin een nieuwe branch: dagobert-duck vanuit master
  • Dagobert is een beetje skeer, en geeft ons 0,01 BTC.
    Pas het hierboven aangegeven donatiebedrag (id=donaties) aan naar: 0,01
  • Commit deze wijziging.

Conflict!

  • Merge de aanpassingen van dagobert-duck naar master.
  • Controleer of het donatie bedrag klopt (refresh de pagina).
  • Merge de aanpassingen van gulle-gever naar master.
  • Conflict!
  • Zorg dat de donaties bij elkaar worden opgeteld.
  • Controleer of het donatiebedrag correct is (refresh de pagina).
  • Commit de merge
  • Push de master branch naar github

5. Een branch rebasen

I are: Baboon

  • Zorg dat je in de master branch zit
  • Begin een nieuwe branch: rebasing
  • Pas de inhoud van het element hierboven (id=animal) aan van Weasel naar: Baboon
  • Commit deze wijziging.

No Span(g)

  • Ga terug naar de master branch
  • Pas het element hierboven aan (id=animal) aan van een span naar een strong
  • Commit deze wijziging.

Rebase!

  • Ga terug naar de rebasing branch
  • Rebase deze branch bovenop de master branch
  • Los het conflict op, controleer het resultaat en maak een commit.
    In het eind resultaat heb je dan een strong met daarin Baboon
  • Merge de aanpassingen van rebase naar master.
  • Push de master branch naar github

6. Versie update

Versiebeheer eindigt natuurlijk altijd met een nieuwe versie.

Opdracht

Zorg dat je in de master branch zit.

  • Verhoog de versie (id=version) met 0.1
  • Commit en push de wijziging
  • Lever de URL van jouw github repository in.
